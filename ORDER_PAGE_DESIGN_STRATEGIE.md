## 📐 PŘED/PO SROVNÁNÍ:

### **URGENCY:**
```

PO:
• Kompaktní bar nahoře v checkout sekci
• Part of checkout flow
• Logicky spojený s formulářem
• Vizuálně konzistentní

→ LEPŠÍ FLOW! ✅
```

---

### **TRUST BADGES:**
```
PŘED:
• Samostatná sekce (bg-gray-50)

• 3 oddělené sekce

PO:

→ LOGIČTĚJŠÍ! ✅

## 🎯 STRATEGICKÉ DŮVODY:

### **1. VISUAL HIERARCHY:**
```
FAQ (šedé)
  ↓
Rekapitulace (fialový gradient) ← "Připraven?"
  ↓
CHECKOUT SEKCE (šedé) ← jediná akce!
  │
  ├─ Urgency (oranžový bar) ← tlak
  ├─ Trust (3 boxy) ← důvěra
  └─ Formulář (bílý card) ← akce

→ JASNÝ FLOW K AKCI! 🎯
```

---

### **2. PSYCHOLOGICAL FLOW:**
```
1. FAQ - "Mám ještě otázky?"
   → Vyřešíme všechny pochybnosti

2. Rekapitulace - "Co vlastně dostanu?"
   → Připomeneme value (90 min, strategie, plán)
   → Připomeneme cenu (4.999 Kč lifetime)
   → Připomeneme záruku (14 dní)

3. CHECKOUT - "Okay, kupuji!"
   → Urgency: "Musím teď!" (countdown)
   → Trust: "Je to bezpečné" (lock/shield)
   → Formulář: "Vyplním a hotovo"

→ LOGICKÝ TOK ROZHODOVÁNÍ! 🧠
```

---

### **3. DESIGNOVÁ KONZISTENCE:**
```
PRAVIDLO:
• Barevné gradients = storytelling sekce
• Bílé pozadí = content sekce
• Šedé pozadí = funkční sekce (FAQ, checkout)
• Bílé cardy = důležité elementy

APLIKACE:
✅ Hero = indigo gradient (dojem)
✅ Problémy = gradient boxy (emoce)
✅ Features = bílé (content)
✅ FAQ = šedé (funkční)
✅ Rekapitulace = fialový gradient (story)
✅ Checkout = šedé pozadí (funkční)
   ├─ Urgency = oranžový bar (důležité!)
   ├─ Trust = bílé boxy (elementy)
   └─ Formulář = bílý card (hlavní akce!)

→ KONZISTENTNÍ SYSTÉM! 🎨
```

### **2. REDUCED COGNITIVE LOAD:**
```
PŘED:
"Velký oranžový pruh" (co to je?)
  ↓
